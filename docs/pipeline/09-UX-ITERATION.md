# ⑨ UX 이터레이션 — 아이콘 & 주스 레이어

> 데스크탑 전환(①~⑧) 이후 첫 기능 이터레이션: 앱 아이콘 + UI/UX 폴리싱 + 재미 요소.
> 원칙 유지: **엔진 무변경**(84 테스트 그대로), 웹 빌드 병행 유지.

## 1. 앱 아이콘 (요구 1)

- **생성 방식**: 에셋 없이 **코드로 파라메트릭 드로잉** — Electron 오프스크린 캔버스가 크기별(16~1024px)로 새로 그림 → 축소 블러 없음. 프로젝트의 "shape-and-code" 철학 유지.
- **구성**: `scripts/icon-electron.cjs`(드로잉) + `scripts/make-icon.mjs`(오케스트레이션: PNG → macOS `iconutil` → icns). `npm run icon`으로 재생성.
- **디자인**: 게임 비주얼 언어 그대로 — 다크 징크 필드 + 희미한 그리드, 시안 점령 영역과 네온 트레일, 플레이어 다이아몬드, 푸크시아 헥스 보스, 옐로 스파크.
- **연결**: `electron-builder.yml`(mac icns / win·linux png, ico는 패키징 시 자동 파생), dev 실행 시 dock/창 아이콘, 패키징 번들 내 `icon.icns` 확인 완료.

## 2. 주스 레이어 아키텍처 (요구 2)

핵심 결정: 엔진은 이벤트를 발행하지 않으므로 **프레임 단위 상태 디핑**으로 이벤트를 유도한다.

```
engine (불변) ──getState()──► watcher.scan()  ──이벤트──►  fx (오버레이 캔버스) + audio (WebAudio 신스)
                              gridVersion↑=claim, lives↓=death,
                              items[i].collected↑=pickup, boss.alive↓=kill,
                              sparks.length↑=warn, status 전이=화면 전환음
```

- **fx 캔버스**: 게임 캔버스 위에 겹친 pointer-events-none 두 번째 캔버스. 렌더러(`renderer.ts`) 무변경.
- **화면 흔들림**: 두 캔버스를 감싼 래퍼에 CSS transform — 캔버스 재드로잉 비용 0.
- **오디오**: 전부 오실레이터/노이즈 합성(에셋 0). 첫 키 입력에서 unlock(웹 autoplay 정책 대응), `M` 토글은 localStorage 영속.

## 5. 2차 폴리싱 (아이템 아이콘 + 이동 이펙트)

**아이템 문자 → 벡터 아이콘** (`renderer.ts`):

| 코드 | 글리프 | 색 |
|---|---|---|
| T | 눈결정(6-spoke + V틱) | ice blue `#7dd3fc` |
| S | 번개 볼트 | fuchsia `#f0abfc` |
| L | 조준경(링+틱+코어) | cyan `#67e8f9` |
| P | 5각 별 | gold `#fde047` |
| C | 8방향 버스트 | emerald `#6ee7b7` |

박스 테두리도 아이템 색으로 통일, 위상차 부유(bob) + 글로우 펄스 애니메이션 추가.

**이동 이펙트** (`fx.ts`/`watcher.ts`):
- **잔상(ghost)**: 이동 중 다이아몬드 아웃라인 애프터이미지 — 드로잉은 푸크시아(고밀도 35ms), 프런티어 이동은 시안(60ms).
- **커팅 스파크**: 어둠을 가르는 헤드에서 푸크시아/백색 엠버 연속 방출(40ms 간격).
- **트레일 점화**: 커밋 순간 직전 트레일 배열 참조를 보관(엔진이 배열을 *교체*하므로 안전) → 경로 전체에 시안 스파클 스윕. 사망/소비 후 참조 리셋으로 오발 방지.

**조작 매끄러움 진단**: 입력은 60Hz 고정 스텝으로 샘플링되고 플레이어 위치는 float(픽셀 보간)라 60Hz 디스플레이에서 이미 프레임 정확·부드러움. 잔상/스파크는 체감 유동성을 올리는 레이어. 120Hz+ 디스플레이 대응(렌더 보간)은 후속 후보.

## 6. 3차 폴리싱 (BGM 교체 + 리사이즈)

**BGM: 앰비언트 패드 → 칩튠 시퀀서** (`audio.ts`):
- 기존 지속음 패드(디튠 트라이앵글+LFO)는 음악이라기보다 험 — 사용자 피드백으로 교체.
- 새 구현: **118 BPM, A minor 4마디 루프(Am→F→C→G)**, 16분음표 그리드.
  - 스퀘어 베이스(8분 드라이빙, 마디 끝 5도/옥타브 워크업)
  - 트라이앵글 아르페지오 리드(2옥타브 위 코드톤, 16스텝 컨투어)
  - 드럼: 사인 드랍 킥(1·3박), 밴드패스 노이즈 스네어(2·4박), 하이패스 햇(8분, 오프비트 액센트)
- WebAudio **룩어헤드 스케줄링**(30ms 인터벌, 120ms 선행 큐) — 샘플 정확 타이밍.
- 일시정지 시 음악 정지/재개, 뮤트 해제 시 자동 복귀(`musicWanted` 플래그).

**창 리사이즈** (`main.ts` + `useFitScale`):
- `resizable/maximizable: true`, 최소 560×500.
- 렌더러는 고정 디자인(768×576 캔버스+HUD+푸터)을 `transform: scale()`로 뷰포트에 맞춤 — **내부 해상도·시뮬레이션 불변**, 프레젠테이션만 스케일. 웹 빌드도 동일 적용.
- 검증: 600×520(축소)·1400×1000(확대) 스크린샷 — 잘림/스크롤 없음.

## 7. 이터레이션 중 조정

- ESLint에 CJS 스크립트 블록 추가(`scripts/**/*.cjs`): Electron 엔트리는 require/__dirname이 관례.
- `vite-plugin-electron` 관련 잔여 없음 — 아이콘 생성도 esbuild 파이프라인과 동일하게 Electron 바이너리 직접 구동.
